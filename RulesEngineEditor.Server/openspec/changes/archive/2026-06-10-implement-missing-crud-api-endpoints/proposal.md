## Why

The RulesEngineEditor frontend requires full CRUD operations for workflow definitions and test scenarios to enable users to create, edit, delete, and manage their rules and scenarios through the editor interface. Currently only read-all, dry-run, and scenario-create endpoints exist—the frontend falls back to localStorage mock data for the missing operations, resulting in a degraded user experience and data that is not persisted.

## What Changes

- **New workflow CRUD endpoints**: GET /api/Rules/{id}, POST /api/Rules, PUT /api/Rules/{id}, DELETE /api/Rules/{id}
- **New scenario CRUD endpoints**: GET /api/Rules/scenarios (with optional workflowId filter), GET /api/Rules/scenarios/{id}, PUT /api/Rules/scenarios/{id}, DELETE /api/Rules/scenarios/{id}
- **New repository methods**: CreateWorkflowAsync, UpdateWorkflowAsync, DeleteWorkflowAsync, GetScenariosAsync, GetScenarioByIdAsync, UpdateScenarioAsync, DeleteScenarioAsync
- **New DTO records**: WorkflowDefinitionResponse (full detail), CreateWorkflowRequest, UpdateWorkflowRequest, ScenarioResponse, UpdateScenarioRequest
- **OpenAPI spec updates**: Both v1.yaml and v1.json updated with new endpoints and schemas
- **Unit tests**: Repository method tests in UnitTests project
- **Integration tests**: Controller endpoint tests in InfrastructureTests project
- **No breaking changes**: All existing endpoints and DTOs remain unchanged

## Capabilities

### New Capabilities
- `workflow-crud`: Full create, read, update, delete operations for workflow definitions including JSON content validation and unique constraint enforcement
- `scenario-crud`: Full create, read, update, delete operations for test scenarios with optional workflowId filtering and JSON validation

### Modified Capabilities
- `rules-engine-api`: NEW endpoints added for workflow and scenario CRUD (GET/POST/PUT/DELETE), extending the existing GET /api/rules and POST endpoints
- `rules-engine-repository`: NEW methods added to IRulesRepository and RulesRepository for workflow persistence (create/update/delete) and scenario queries (list/get by id/update/delete)

## Impact

- **Controllers/RulesController.cs**: Added 8 new action methods + 5 new DTO records
- **Infrastructure/Repositories/IRulesRepository.cs**: Added 7 new method signatures
- **Infrastructure/Repositories/RulesRepository.cs**: Added 7 new method implementations using EF Core
- **Api/v1.yaml, Api/v1.json**: Added new endpoint paths, request/response schemas
- **Testing/RulesEngineEditor.Server.UnitTests**: New test classes for repository methods
- **Testing/RulesEngineEditor.Server.InfrastructureTests**: New test classes for controller endpoints
- **No entity model changes**: WorkflowDefinitions and WorkflowTestScenarios are auto-generated by EF Core Power Tools and remain unmodified
